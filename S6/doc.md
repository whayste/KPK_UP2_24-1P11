Вариант 6: Сервис специальностей

## ER-диаграмма
![ER-диаграмма](erd.png)

### Описание структуры данных
Архитектура базы данных спроектирована в третьей нормальной форме (3NF):
1. **Сущность SPECIALTY (Специальности):** Содержит явный первичный ключ `id`, уникальный код специальности (`code`), наименование (`name`) и системный флаг активности записи (`is_active`) для реализации механизма мягкого удаления.
2. **Сущность FGOS (ФГОС):** Хранит реестр уникальных кодов стандартов ФГОС (`code`) и первичный ключ `id`.
3. **Сущность SPECIALTY_FGOS (Транзитивная таблица):** Реализует связь типа «многие-ко-многим» между специальностей и ФГОС через составной первичный ключ, состоящий из внешних ключей `specialty_id` и `fgos_id`. Поля внешних ключей имеют жесткий запрет на хранение значений NULL.

## 1. Манипулирование данными: SPECIALTY

### Добавить <сущность>
**Информация требуемая для создания:**


| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| code | Уникальный код специальности | Да | String | Unique, Not Null | отсутствует |
| name | Наименование специальности | Да | String | Not Null | отсутствует |

**Перечислить уникальные комбинации параметров:** Поле `code` должно быть уникальным во всей таблице.

**Информация возвращаемая в случае удачного создания:**


| Параметр | Тип |
| :--- | :--- |
| id | Integer |
| code | String |
| name | String |
| is_active | Boolean |

### Изменить <сущность> по ID
**Информация требуемая для изменения:**


| Параметр | Пояснение | Обязательность | Тип | Ограничение |
| :--- | :--- | :--- | :--- | :--- |
| id | Идентификатор изменяемой записи | Да | Integer | Первичный ключ |
| code | Уникальный код специальности (Изменение не производится, поле защищено от перезаписи) | Нет | String | не подлежит изменению |
| name | Новое наименование специальности | Нет | String | - |

**Информация возвращаемая в случае удачного изменения:**


| Параметр | Тип |
| :--- | :--- |
| id | Integer |
| code | String |
| name | String |
| is_active | Boolean |

---

## Удаление <сущность> по ID

Вернёт `True`, если `<сущность>` была закрыта (удалена), иначе вернёт `False`.
Фактически запись из БД не удаляется, а реализуется через булевое поле `is_active`.

### Получить список <сущностей> по заданным параметрам
**Информация требуемая для получения списка:**


| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| limit | Ограничение на количество выводимых записей для пагинации | Нет | Integer | limit > 0 | 10 |
| offset | Смещение относительно начала выборки для пагинации | Нет | Integer | offset >= 0 | 0 |
| search | Текстовая строка для фильтрации результатов по частичному совпадению в полях code или name | Нет | String | - | - |
| sort_by | Имя поля, по которому будет отсортирован список результатов (доступные поля для сортировки: code, name) | Нет | String | - | - |

**Информация возвращается в виде списка со следующими столбцами:**


| Параметр | Тип |
| :--- | :--- |
| id | Integer |
| code | String |
| name | String |
| is_active | Boolean |
